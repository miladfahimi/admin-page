Domain Roles
Domain Role

Provider

  • Tennis club or venue that supplies services (courts, coaching).
  • Owns catalog, pricing, and maintenance policies.
  • Works closely with Provider & Court Management and Analytics.
Key Responsibilities
  • Publish availability for each service.
  • Respond to booking notifications and adjustments.
  • Maintain facility readiness and quality.
Domain Role

Service

  • Bookable asset owned by a provider (court, lesson, equipment).
  • Contains capacity rules, pricing models, and prep buffers.
  • Managed through Provider & Court Management, consumed by Booking.
Domain Notes
  • Value objects define schedules and time slots.
  • Policies enforce downtime, maintenance, and overbooking rules.
  • Events propagate to Analytics and Notifications.
Domain Role

Customer

  • Player who discovers providers and books services.
  • Interacts through the Customer Web App and receives notifications.
  • Subject to booking policies, payments, and loyalty programs.
Experience Touchpoints
  • Identity & Access for authentication.
  • Booking and Payment flows for reservations.
  • Notification service for reminders and updates.
Domain Role

Admin

  • Platform operator managing providers, services, and compliance.
  • Uses the Admin Panel for oversight and configuration.
  • Coordinates incident response and rollout plans.
Key Capabilities
  • Approve provider onboarding and catalog changes.
  • Monitor analytics and enforce policy adherence.
  • Manage infrastructure transitions (Compose ‚Üí Kubernetes).
Experience Layer
Frontend

Customer Web App (Angular)

  • Discover providers, book services, manage reservations, and view history.
  • Integrates with BFF layer for optimized APIs.
  • Provides payment, loyalty, and notification workflows.
Key UI concerns
  • Calendar components for slot selection.
  • Responsive layout for mobile booking.
  • Real-time updates via WebSocket or SSE.
Frontend

Admin Panel (Angular)

  • Manage providers, services, schedules, pricing, and promotions.
  • Provides dashboards for utilization analytics.
  • Role-based access across providers, their staff, and platform admins.
Key UI concerns
  • Rich data tables with filtering and export.
  • Integrations with analytics microservice.
  • Audit trails and activity feeds.
Edge & Orchestration
Gateway

API Gateway / BFF

  • Routing, rate-limiting, and API composition.
  • Authentication handoff to Identity service.
  • GraphQL/REST aggregation for frontends.
Core Service

Identity & Access

  • Account provisioning, SSO, and role management.
  • Issues tokens via OAuth/OIDC.
  • Supports customer, provider, and admin personas with scoped roles.
Domain Notes
  • Bounded Context: Identity.
  • Aggregates: User, Role, Session.
  • Integrates with audit logging for compliance.
Core Booking Domain
Core Service

Provider & Court Management

  • Handles provider profiles, service inventory, availability, and maintenance windows.
  • Tracks provider policies, preparation buffers, and blackout periods per service.
  • Exposes events for scheduling updates.
DDD Focus
  • Bounded Context: Provider & Court Management.
  • Aggregates: Provider, ProviderService, TimeSlot, MaintenanceWindow.
  • Policies for maintenance and setup time.
Core Service

Booking Service

  • Validates booking rules and conflicts between customers and provider services.
  • Supports waitlists, cancellations, reassignments, and provider overrides.
  • Publishes provider/customer domain events for notifications and analytics.
DDD Focus
  • Bounded Context: Booking.
  • Aggregates: Booking, ReservationPolicy, ProviderServiceRef, TimeSlot.
  • Value Objects tie customer demand, provider capacity, availability, and payments.
Core Service

Pricing & Payment

  • Calculates provider-specific pricing, discount models, and payout schedules.
  • Manages payment intents, settlement flows, and third-party gateways.
  • Tracks settlement and invoice state.
Integrations
  • External payment processors (Stripe, Adyen).
  • Ledger or accounting systems.
  • Fraud detection services.
Core Service

Notification Service

  • Subscribes to booking and payment events.
  • Sends provider and customer email, SMS, and push notifications.
  • Template and channel management.
Channels
  • SMS gateway integrations.
  • Email providers (SendGrid, SES).
  • Mobile push via Firebase/APNs.
Core Service

Analytics & Reporting

  • Consumes domain events for utilization metrics.
  • Provides dashboards and exports for admins and providers.
  • Supports experimentation and cohort analysis.
Data Strategy
  • Event sourcing or CDC pipelines.
  • Aggregated data warehouse (BigQuery, Snowflake).
  • BI tooling integration.
